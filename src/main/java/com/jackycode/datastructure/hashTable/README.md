1. 情景：有個員工到公司報到，要求儲存該員工的信息(id、性名、年齡...)，當輸入id時可以查到，不使用數據庫，盡量節省內存。這時就可以使用Hash Table。  
在早些年代，沒有緩存產品，就要自己寫緩存實現

    * 數組＋二叉樹
